# Security Policy

## Scope and status

TactileRide is at a documentation-only bootstrap stage. No firmware or release
artifact is currently supported. Security reports remain valuable because early
design and repository-process issues are easier to address before prototypes
exist.

## Reporting a vulnerability

Do not publish a suspected vulnerability, sensitive configuration, or private
personal data in a public issue. If GitHub private vulnerability reporting is
enabled for this repository, use it. Otherwise, contact a project maintainer
through a private channel published by the repository owner and include:

- A concise description and affected path or revision.
- Reproduction or observation steps.
- Impact and any safe mitigation you identified.
- Whether public disclosure has occurred or is planned.

## Response expectations

Response timing and support windows are not yet committed. Maintainers should
acknowledge the report when possible, assess it privately, document the fix or
decision, and coordinate disclosure with the reporter where appropriate.
