# Licence Candidates and Status

## Current status

No final licence text has been adopted for TactileRide. This directory documents
the planned multi-licence structure; it does not itself grant rights or make a
compatibility claim. Do not assume that an unlicensed artifact is released under
any candidate below.

## Candidate structure

| Artifact class | Candidate to evaluate | Reason for consideration | Decision state |
| --- | --- | --- | --- |
| Firmware and software tools | Apache-2.0 or MIT | Widely used permissive software candidates; Apache-2.0 includes an explicit patent grant. | Open |
| Electronics and mechanical design | CERN-OHL-S v2 or CERN-OHL-P v2 | Open-source-hardware candidates with different reciprocity obligations. | Open |
| Documentation and content | CC BY 4.0 | Attribution-based documentation/content candidate. | Open |

## Review needed before adoption

1. Verify current official licence texts and their scope.
2. Review compatibility among source code, generated artifacts, hardware design
   files, documentation, and third-party contributions.
3. Define file-level notices and contribution terms.
4. Obtain project-governance approval and add the exact texts.
5. Supersede ADR 0007 with the adopted decision.

Until those steps occur, contributors should ask before contributing material
whose licensing terms cannot be reconciled with the project.
